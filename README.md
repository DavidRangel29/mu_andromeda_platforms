*   **Permissions:** What users are allowed to do with the software (e.g., run it, modify it, distribute it).
*   **Conditions:** Requirements that must be met when using the software (e.g., retaining copyright notices, including the license text).
*   **Limitations:** What users are *not* allowed to do, and disclaimers about warranties or liability.

Many open-source projects, including those common in the Android development space, use licenses like the Apache License, MIT License, or GPL (General Public License).
